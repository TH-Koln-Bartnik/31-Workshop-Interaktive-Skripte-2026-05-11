Was Sie heute bauen

Ihr eigenes interaktives Skript live im Netz.

Am Ende des Tages können Sie …

  1. Ein leicht pflegbares Online-Skript erstellen (ein Quarto-Projekt lokal in RStudio anlegen, rendern und über GitHub Pages veröffentlichen),
  2. …das verschiedene Formatierungen automatisch erzeugt (etwa ein automatisches Inhaltsverzeichnis erzeugen und Quellen aus Zotero zitieren),
  3. …Stilelemente an Ihre Vorlieben anpasst (das Skript im TH-Köln-Stil formatieren (_brand.yml, CSS)),
  4. …und den Lernenden erlaubt, mit den Inhalten zu interagieren (ein interaktives Diagramm und ein iFrame-Spiel einbauen),
  5. …und Hilfe durch KI bereitzustellen (einen KI-Tutor verlinken, der die Lernenden im Skript begleitet).

Wählen Sie Ihren Beispielstrang

Damit Sie nicht auch noch Inhalte erfinden müssen, nutzen wir ein durchgehendes Mini-Thema: eine kurze Einführung ins wissenschaftliche Arbeiten. Wählen Sie einen der drei Stränge.

Strang Worum es geht Quelle
A — Wissenschaftliche Einstellung Skepsis, Belege, intellektuelle Demut Bartnik & Palmer (2024), Kap. „Wissenschaftliche Einstellung”
B — Recherche und Quellenqualität Suchen, prüfen, Quellen einordnen Bartnik & Palmer (2024), Kap. „Kritische Informationssuche”
C — Transparent schreiben Methoden offenlegen, Reproduzierbarkeit Bartnik & Palmer (2024), Kap. „Transparent schreiben”

Sie können natürlich auch eigene Inhalte einfügen. Die Wahl ist Geschmackssache. Die technischen Schritte sind in jedem Strang identisch.

Tagesablauf — drei Phasen

Der Workshop ist so strukturiert, dass Sie nicht alle Werkzeuge gleichzeitig lernen müssen. Phase 1 (Vormittag): nur RStudio plus KI-Tutor — alles lokal, kein Internet außer dem Tutor-Tab. Phase 2 (früher Nachmittag): Zotero kommt dazu, immer noch lokal. Phase 3 (später Nachmittag): GitHub und Pages — Ihr Skript geht ins Netz.

09:00–09:30 Auftakt — Vorstellung, Vorerfahrung, Ziele, Tour durch die drei Vorbilder, Strangwahl
09:30–10:15 Kap. 1 — Didaktik — Drei Lerntheorie-Argumente, Lernspiel zur Selbstprüfung
10:15–11:30 Kap. 2 — RStudio + erstes Skript 1 RStudio-Cockpit, Quarto-Buch lokal anlegen, quarto preview im Browser, Tutor-Routine entwickeln
11:30–11:45 Pause

Wie Sie dieses Skript benutzen

Jedes Kapitel öffnet mit einem Lernziel-Kasten („Was Sie nach diesem Kapitel kannst”), gefolgt von einem „Mach mit”-Block — das ist die eigentliche Übung. Dazwischen steht so wenig Text wie möglich. Am Ende jedes Kapitels finden Sie einen Trouble-Shooting-Kasten („Was, wenn …“) und einen fertigen Tutor-Prompt zum Kopieren.

RStudio ist Ihre Arbeitszentrale

Innerhalb von RStudio haben Sie Editor, Terminal (für quarto render und quarto preview), Files-Pane (zum Anlegen von Ordnern und Dateien), Build-Pane (Knopf „Render Book”) und Console (für R-Pakete) auf einem Schirm. Daneben brauchen Sie nur GitHub Desktop zum Versionieren und einen Browser-Tab mit dem Tutor.

Ordnerstruktur Ihres Skripts

Damit Sie immer wissen, wo etwas hingehört (folgt dem Aufbau bei Daniel Lakens):

Ordner Was kommt rein?
parts/ Ihre Kapiteldateien (.qmd)
images/ Logos, Screenshots, statische Grafiken (PNG, JPG, SVG)
interactions/ HTML-Widgets ohne Backend (Lernspiel, Memory, Quiz)
include/ Wiederverwendbare qmd-Schnipsel (per {{< include >}})
data/ CSV/JSON-Daten für interaktive Charts
appendix/ Quickstart, Troubleshooting, Tutor-Prompt

Voraussetzungen

Wenn Sie noch nichts installiert haben, springen Sie zuerst in den Quick-Starter Guide. Dort steht, was Sie in welcher Reihenfolge installieren (Quarto, R, RStudio, GitHub Desktop, Zotero) — mit Screenshots.
