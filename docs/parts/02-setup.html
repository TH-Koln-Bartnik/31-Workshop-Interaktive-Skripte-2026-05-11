2 — RStudio + erstes Skript (lokal)

Was Sie nach diesem Kapitel können

  • die vier Werkzeuge Quarto, YAML, RStudio, HTML in eigenen Worten erklären,
  • ein leeres Quarto-Buch lokal anlegen und im Browser als Live-Vorschau ansehen,
  • den KI-Tutor als ständigen Begleiter im zweiten Browser-Tab nutzen — drei konkrete Anlässe sitzen am Ende dieses Kapitels.

Worum es im Kern geht

Sie schreiben Text in einer Datei. Ein Programm verwandelt diese Datei in eine Webseite, die in Ihrem Browser auf localhost lebt — also nur auf Ihrem Computer, ohne Internet, ohne Server. Eine zweite Datei legt fest, wie das Ergebnis aussieht. Bis zum Mittag bleiben wir in genau dieser lokalen Welt — kein GitHub, keine Veröffentlichung, kein Anmeldekonto. Das senkt die Stolperfallen auf eins: Quarto. So entsteht Routine in der Schreib-Render-Schleife, bevor wir später Versionsverwaltung und Web-Speicherung dazuschalten.

Dass wissenschaftliches Arbeiten von einer reproduzierbaren Render-Logik profitiert, ist seit langem belegt (Bryan, 2018); Quarto bringt diese Logik in die Hochschullehre (Allaire et al., 2024).

Die vier Werkzeuge — in einfachen Worten zuerst

Jeder Begriff klingt in der Fachsprache schwerer als nötig. Hier zuerst die einfache Idee, dann das Detail.

Werkzeug In einem Satz Was es technisch ist
Quarto Programm, das aus einer Textdatei eine Webseite, ein PDF oder Folien macht. Open-Source-Publishing-System, Nachfolger von R Markdown (Allaire et al., 2024).
YAML Einstellungsdatei am Anfang jedes Kapitels — bestimmt Titel, Format, Inhaltsverzeichnis. „YAML Ain’t Markup Language”, strukturiertes Format mit Einrückungen.
RStudio Schreibtisch für alles, was wir heute tun: Editor, Terminal, Files, Build — auf einem Schirm. Integrated Development Environment für R und Quarto, gehört Posit.
HTML Sprache, in der Webseiten geschrieben sind. „HyperText Markup Language”, Tags wie <h1> oder <p>.

JSON, iFrames und GitHub kommen später — heute brauchen wir sie noch nicht.

Woher die Namen kommen — Eselsbrücken zum Merken

Vier Namen, vier kleine Geschichten. Wer die Anekdote kennt, vergisst den Begriff nicht mehr.

Quarto stammt aus dem Lateinischen quartus („vierter”) und bezeichnet seit dem Buchdruck ein Papierformat, das durch zweimaliges Falten eines Bogens entsteht — ein Quartband ist ein Buch im Quartformat. Der Name nickt also der jahrhundertealten Tradition des Publizierens zu, in deren Linie sich dieses Werkzeug stellt.

YAML ist ein selbstironisches rekursives Akronym: „YAML Ain’t Markup Language” — „YAML ist keine Auszeichnungssprache”. Ursprünglich hieß es Yet Another Markup Language; die Erfinder änderten den Sinn, um klarzustellen, dass YAML Daten beschreibt, nicht Dokumente. Solche selbstreferentiellen Wortspiele sind in der Open-Source-Welt eine Tradition (vgl. GNU = GNU’s Not Unix).

RStudio ist nach der Programmiersprache R benannt, die wiederum nach den Vornamen ihrer Erfinder Robert Gentleman und Ross Ihaka heißt — beide an der University of Auckland. Die Firma hinter RStudio nennt sich seit 2022 Posit (lateinisch positum, „gesetzt”), die Software behält den vertrauten Namen.

HTML steht für HyperText Markup Language. „Hypertext” hat Ted Nelson 1965 geprägt — die Idee, dass Texte nicht mehr linear gelesen werden müssen, sondern über Verweise frei verbunden sind. HTML selbst entwickelte Tim Berners-Lee 1990 am CERN als Beilage zum World Wide Web; er wollte Physikern das Teilen von Forschungspapieren erleichtern.

Mach mit — Zuordnungsübung zu den Werkzeugen

Drei Minuten, vier Paare. Ordnen Sie jeden Begriff der Definition in einfachen Worten zu — links Begriff anklicken, rechts Definition anklicken.

Zuordnungsübung öffnen →

Variieren Sie
  1. Fünftes Paar: Öffnen Sie interactions/memory-spiel.html in RStudio, ergänzen Sie ein fünftes Begriffspaar (Vorschläge: „Markdown”, „Render”, „Theme”) inklusive Definition in einfachen Worten.
  2. Selbst-Test: Spielen Sie die Übung zu zweit auf Zeit — wer ordnet alle vier Paare beim ersten Versuch korrekt zu?
  3. Reihenfolge: Schreiben Sie die vier Werkzeuge in der Reihenfolge auf, in der Sie ihnen heute begegnen — von „RStudio öffnen” bis „lokale Webseite im Browser sehen”.

Der Vormittags-Workflow in einem Bild

Zwei Werkzeuge, ein Kreislauf:

RStudio (schreiben + rendern)  →  Browser-Vorschau auf localhost (ansehen)
        ↑                                                            │
        └──────────────────  Tutor im zweiten Tab (helfen)  ─────────┘

Versionsverwaltung und Web-Veröffentlichung kommen am Nachmittag in Kapitel 8 dazu. Jetzt geht es um die Schreib-Render-Schleife.

RStudio als Arbeitszentrale — vier Bereiche

Sie arbeiten fast ausschließlich in RStudio. Vier Bereiche reichen für den Vormittag:

Bereich Wo? Wofür?
Editor oben links .qmd-Dateien schreiben
Terminal unten links, Tab neben Console quarto render, quarto preview
Files unten rechts neue Ordner und Dateien anlegen, Dateien öffnen
Build oben rechts Knopf „Render Book” als Klick-Alternative

Drei nützliche Shortcuts:

  • Strg+Shift+K — die aktuell geöffnete .qmd-Datei einzeln rendern (für schnelles Vorschauen einzelner Kapitel).
  • Strg+Shift+B — komplettes Buch rendern (entspricht dem Build-Knopf).
  • Alt+Shift+R — neuen Terminal-Tab öffnen, falls keiner sichtbar ist.

Falls Ihr Terminal-Tab fehlt: Tools → Terminal → New Terminal. Voreinstellung auf Windows ist PowerShell, einstellbar unter Tools → Global Options → Terminal. Quarto-Befehle funktionieren in jeder Shell.

Mach mit — vom leeren Ordner zur ersten Vorschau

Halten Sie sich an diese Reihenfolge. Bei Problemen: Tutor fragen oder zur Trouble-Shooting-Box am Ende springen.

Schritt 1 — neues Projekt in RStudio

  1. RStudio öffnen → File → New Project → New Directory → Quarto Book.
  2. Ordnernamen wählen, z. B. mein-erstes-skript.
  3. Speicherort wählen — irgendwo in Ihrem Documents-Ordner.
  4. Create Project.

RStudio legt automatisch eine Mini-Buchstruktur an (_quarto.yml, index.qmd, ein paar Beispielkapitel).

Schritt 2 — Render und Preview (in RStudio)

Im Terminal-Tab (unten neben Console):

quarto preview

Im Browser öffnet sich http://localhost:4XXX mit einer Live-Vorschau. Jede Änderung an einer .qmd-Datei führt zu sofortigem Reload. Alternative ohne Tippen: Build-Pane oben rechts → Knopf „Render Book” (rendert das ganze Buch einmal, ohne Live-Reload).

Schritt 3 — Beispielkapitel auf Ihren Strang anpassen

Öffnen Sie intro.qmd und ersetzen Sie den Beispielinhalt durch eine Mini-Einleitung zu Ihrem gewählten Strang. Drei Vorlagen, je nach Strangwahl:

  • Strang A — Wissenschaftliche Einstellung
  • Strang B — Recherche und Quellenqualität
  • Strang C — Transparent schreiben
# Wissenschaftliche Einstellung

Wissenschaftliche Einstellung heißt, Aussagen nicht zu glauben, sondern
zu prüfen — und sich selbst genauso zu prüfen wie andere. Drei Haltungen
gehören dazu: Skepsis, intellektuelle Demut, Bereitschaft zur Korrektur.
Quelle: Bartnik & Palmer (2024).
# Recherche und Quellenqualität

Quellen unterscheiden sich nicht nur darin, ob sie wahr oder falsch sind,
sondern darin, wie nachprüfbar sie sind. In diesem Kapitel: drei Filter,
mit denen Sie eine Quelle in 30 Sekunden grob einordnen.
Quelle: Bartnik & Palmer (2024).
# Transparent schreiben

Transparenz heißt, dass jeder Leser nachvollziehen kann, wie Sie zu Ihren
Ergebnissen gekommen sind — welche Daten, welche Methoden, welche
Annahmen. In diesem Kapitel: drei Bausteine eines transparenten
Methodenteils. Quelle: Bartnik & Palmer (2024).

Schritt 4 — Ordnerstruktur anlegen (in RStudio)

Im Files-Pane (unten rechts) klicken Sie auf New Folder und legen nacheinander an: images/, interactions/, include/, data/. Das sind die Ordner für Ihre späteren Bilder, HTML-Widgets, eingebundene Schnipsel und Datendateien — die Aufteilung folgt der Konvention, die Lakens (2022) in seinem Skript verwendet.

Was nicht heute Vormittag passiert

Versionsverwaltung mit Git, GitHub-Konto, GitHub Desktop, GitHub Pages — das alles kommt in Kapitel 8 — Veröffentlichen mit GitHub Pages ganz am Ende des Tages dran. Bis dahin lebt Ihr Skript zufrieden auf Ihrem Computer.

KI-Tutor als ständiger Begleiter — drei Anlässe

Der Tutor lebt in einem zweiten Browser-Tab und antwortet auf alles, was im Workshop-Kontext steckt. Damit aus dem „könnte ich mal fragen” Routine wird, drei konkrete Anlässe:

Tutor-Anlass 1 — YAML-Fehler verstehen

Provozieren Sie absichtlich einen YAML-Fehler in _quarto.yml: löschen Sie einen Doppelpunkt oder ändern Sie die Einrückung von zwei auf drei Leerzeichen. Render mit Strg+Shift+B. Kopieren Sie die Fehlermeldung in den Tutor-Tab und fragen Sie: „Was bedeutet diese Quarto-Fehlermeldung in einfachen Worten? [Meldung einfügen]“. Korrigieren Sie nach der Antwort, render erneut.

Tutor öffnen →

Tutor-Anlass 2 — Beispielinhalt erfinden lassen

Im Tutor: „Ich schreibe ein Quarto-Skript zum Strang [A/B/C]. Schlagen Sie mir drei kurze Sätze vor, die ich als Einstieg in mein erstes Kapitel verwenden kann. Erklärung in einfachen Worten zuerst, dann der konkrete Text.”. Übernehmen Sie die Sätze in intro.qmd und render.

Tutor-Anlass 3 — Render-Output prüfen lassen

Kopieren Sie das Quarto-Render-Log (alle Zeilen, die im Build-Pane oder Terminal beim Rendern erscheinen) in den Tutor: „Hier mein Render-Log. Sehen Sie Warnungen, die ich beheben sollte? Erklärung pro Warnung in einem Satz.”. Wenn der Tutor Warnungen identifiziert, beheben Sie eine.

Variieren Sie weiter
  1. Titel ändern: Öffnen Sie _quarto.yml und ändern Sie den Buchtitel. Render — wo erscheint der neue Titel überall (Sidebar, Browser-Tab, Footer)?
  2. Eigenes Kapitel anlegen: Legen Sie eine zweite .qmd-Datei an (z. B. parts/mein-thema.qmd) mit zwei Sätzen Inhalt. Tragen Sie sie in _quarto.yml unter chapters: ein. Render und prüfen.
  3. Footer anpassen: Ergänzen Sie unter book.page-footer Ihren eigenen Namen und das aktuelle Jahr. Render.
  4. Theme tauschen: Ersetzen Sie unter format.html.theme cosmo durch ein anderes Bootstrap-Theme (z. B. flatly oder journal). Render und vergleichen.
Was, wenn …
  • YAML-Fehler beim Rendern? → Einrückungen prüfen (zwei Leerzeichen, keine Tabs). Doppelpunkte und Anführungszeichen ebenfalls.
  • quarto: command not found? → Quarto-Installation prüfen mit quarto --version. Wenn leer: neu installieren von https://quarto.org/docs/get-started/.
  • http://localhost:4XXX lädt nicht? → Im Terminal quarto preview erneut starten; manchmal hängt der vorherige Prozess. Strg+C im Terminal beendet ihn sauber.
  • RStudio findet kein Terminal? → Tools → Terminal → New Terminal, oder Shortcut Alt+Shift+R.
