6 — Slides mit Quarto (optional)

Optional, für Fortgeschrittene

Dieses Kapitel ist nicht Teil der gemeinsamen Übung. Wer mit Kapitel 1–5 früher fertig ist, baut hier eigene Folien. Vorlage und Inspiration: Békés (2025), Course slides for “Data Analysis with AI”.

Was Sie nach diesem Kapitel können

  • aus einem .qmd-Kapitel einen Foliensatz mit Quarto Reveal.js erzeugen,
  • Stichpunkte schrittweise einblenden (Appear-Effekt wie in PowerPoint),
  • Bilder und Diagramme in Folien einbetten — auch in zwei Spalten neben dem Text,
  • die TH-Köln-Farben auf Slides übertragen.

Worum es im Kern geht

Folien sind nicht der bessere Skripttext, sondern ein eigener Modus — sie unterstützen die mündliche Erläuterung. Mayer (2021) zeigt: Folien wirken am stärksten, wenn sie Bild plus wenig Text kombinieren und Inhalte schrittweise enthüllen. Genau das liefert Quarto Reveal.js mit zwei Konstrukten: incremental (Listen punktweise) und ::: {.fragment} (beliebige Blöcke einzeln).

Schnellbau in vier Schritten

  1. Neue Datei slides/template-vorlesungsfolien.qmd anlegen — oder die vorhandene Vorlage kopieren.
  2. YAML-Header ergänzen (siehe unten).
  3. Inhalt strukturieren: eine ##-Überschrift = eine Folie.
  4. Im RStudio-Terminal rendern: quarto render slides/template-vorlesungsfolien.qmd oder Knopf Render im Editor.

YAML-Header für Reveal.js

---
title: "Vorlesungsfolien — Vorlage"
author: "Ihr Name"
format:
  revealjs:
    theme: [default, ../styles.scss]
    slide-number: true
    chalkboard: true
    incremental: false
    logo: ../images/th-koeln-logo.png
    footer: "Workshop · TH Köln"
    transition: fade
---

Wichtig: theme: [default, ../styles.scss] zieht Ihre TH-Köln-Farben auch auf die Folien.

Bausteine — sieben Techniken zum Übernehmen

1. Eine Folie pro ##-Überschrift

## Eine Folie pro Überschrift

Stichpunkte direkt darunter, Bilder, Code — was Sie wollen.

2. Section-Divider (großer Trenner zwischen Themenblöcken)

# Block 2 — Stichpunkte

Ein einfaches # (statt ##) erzeugt eine Übersichts-/Trennfolie.

3. Stichpunkte mit Appear-Effekt — einer nach dem anderen

::: incremental
- Punkt eins
- Punkt zwei
- Punkt drei
:::

Im Vortrag: erst der erste Punkt, dann auf Klick (Pfeiltaste, Leertaste oder Klicker) der zweite, der dritte.

4. Beliebige Blöcke einzeln einblenden

::: {.fragment}
**Beobachtung:** Lernende vergessen 70 % innerhalb einer Woche.
:::

::: {.fragment}
**Konsequenz:** Spaced Repetition statt Frontalvortrag.
:::

::: {.fragment}
**Konkret heute:** Mini-Quizze in jedem Skriptkapitel.
:::

Drei Klicks, drei Erkenntnis­schritte — perfekt für Argumentationsketten.

Die komplette Vorlage zum Kopieren

Acht Beispielfolien — Titel, Section-Divider, Bullets statisch und mit Appear, Bild ganzseitig, Bild mit Fragment, zwei Spalten, Code-Folie mit Zeilen-Highlighting, Schlussfolie. Liegt im Projekt unter:

slides/template-vorlesungsfolien.qmd

Im RStudio-Files-Pane öffnen, ansehen, einzelne Folien herauspicken und in eigene Foliensätze übernehmen.

Mach mit — eigene Mini-Foliensequenz

Bauen Sie eine eigene Drei-Folien-Sequenz zu einem Argument aus Ihrem Strang. Vorgehen:

  1. Folie 1 — Frage: Eine zentrale Frage als Headline (## Was ist …?).
  2. Folie 2 — Argument: Drei Punkte mit ::: incremental, die einer nach dem anderen erscheinen.
  3. Folie 3 — Auflösung: Eine Quintessenz, mit ::: {.fragment .highlight-current-red} markiert.

Render mit Strg+Shift+K oder im Terminal: quarto render slides/meine-sequenz.qmd. Im Browser durchklicken (Pfeiltaste / Leertaste).

Variieren Sie
  1. Drei-Spalten-Layout: Erweitern Sie das Zwei-Spalten-Beispiel um eine dritte Spalte (width="33%" jeweils). Wie wirkt das visuell — überladen oder strukturiert?
  2. Bild + Text mit Fragment: Lassen Sie ein Bild und einen Erklärtext gleichzeitig auf Klick erscheinen — packen Sie beide in einen ::: {.fragment}-Block.
  3. Speaker-Notes prüfen: Drücken Sie während des Foliendurchlaufs die Taste „S” — die Sprecher­ansicht öffnet sich in einem zweiten Fenster mit Notizen und Vorschau der nächsten Folie.
  4. Eigenes Theme: Tauschen Sie theme: [default, ../styles.scss] gegen ein dunkles Theme: theme: [dark, ../styles.scss]. Render und vergleichen.

TH-Köln-Brand auf Slides

Die _brand.yml aus Ihrem Skript wird auch von Reveal.js gelesen, sobald Sie sie in den Reveal-YAML einbauen:

format:
  revealjs:
    brand: ../_brand.yml
    theme: [default, ../styles.scss]
Was, wenn …
  • Pfeiltaste blättert über Fragmente weg statt sie einzeln zu zeigen? → incremental: true im YAML setzen oder einzelne Listen mit ::: incremental umhüllen.
  • Bild wird nicht angezeigt? → Pfad relativ zur Slides-Datei: aus slides/foo.qmd zeigt ../images/bild.png ins richtige Verzeichnis.
  • Diagramm aus R-Chunk erscheint nicht? → Pakete prüfen (install.packages("plotly")), und am Chunk #| echo: false setzen, sonst stört der Code-Block die Folienaufteilung.
  • Folien zu textreich? → Faustregel: max. 6 Zeilen pro Folie, eine Idee pro Folie.

Vorschlag-Prompt für dieses Kapitel:

Ich will aus meinem bestehenden Quarto-Kapitel einen Foliensatz mit Reveal.js machen. TH-Köln-Brand soll erhalten bleiben. Schlagen Sie mir die einfachste Aufteilung vor: Welche Abschnitte werden eine Folie, welche mehrere? Welche Stichpunkte sollen mit Appear-Effekt erscheinen? Geben Sie mir den YAML-Header und ein bis zwei Beispielfolien.

End-Test

Drei bis fünf Minuten. Ziehen Sie jeden Begriff auf die passende Wirkung, dann auf „Auflösung anzeigen” klicken — Sie erhalten Feedback pro Eintrag.
