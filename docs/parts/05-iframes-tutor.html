7 — iFrames, Spiele und KI-Tutor

Was Sie nach diesem Kapitel können

  • ein interaktives Quiz oder Spiel als iFrame in Ihr Skript einbinden,
  • den Unterschied zwischen statischem HTML, H5P und externer Einbettung benennen,
  • einen KI-Tutor als Link einbauen, der die Lernenden begleitet — didaktisch begründet.

Worum es im Kern geht

Aktivität schlägt Konsum. Lernende, die zwischendurch etwas tun — antworten, ziehen, ausprobieren — behalten mehr und verstehen besser (Fiorella & Mayer, 2016; Karpicke & Blunt, 2011). Interaktive Elemente sind das Werkzeug, das diese Aktivität in einem statischen Skript ermöglicht. Bei generativer KI als Tutor kommt eine zweite Chance hinzu: ein adaptiver Gesprächspartner, der auf die jeweilige Verständnisstufe reagiert (Kasneci et al., 2023; Mollick & Mollick, 2023).

Live-Beispiel 1 — Drag-and-Drop-Lernspiel als iFrame

Hier zuerst, was am Ende dabei herauskommen kann. Probieren Sie es direkt aus — drei Prinzipien auf drei Beispiele ziehen, dann „Auflösung anzeigen”:

Code zum Übernehmen (an gewünschter Stelle in Ihr .qmd-Kapitel einfügen):

<iframe src="../interactions/lernspiel-kapitel1.html"
        width="100%" height="900" frameborder="0"
        style="border: 1px solid #e6e6ea;">
</iframe>

Was die Attribute bedeuten:

Attribut Wozu?
src Pfad zur HTML-Datei, relativ zum Kapitel (../ geht eine Ebene hoch)
width="100%" füllt die Spaltenbreite
height="900" feste Höhe in Pixeln — anpassen, bis nichts abgeschnitten ist
frameborder="0" kein hässlicher 3D-Rahmen
style="border: …" dezenter eigener Rahmen in TH-Grau

Live-Beispiel 2 — Zuordnungs-Übung als iFrame

Dasselbe iFrame-Prinzip, anderer Übungstyp — vier Begriffe links, vier Definitionen rechts, Klick-Paar-Mechanik:

Code zum Übernehmen:

<iframe src="../interactions/memory-spiel.html"
        width="100%" height="600" frameborder="0"
        style="border: 1px solid #e6e6ea;">
</iframe>

Damit das funktioniert, müssen die HTML-Dateien tatsächlich im Ordner interactions/ liegen — und in _quarto.yml muss unter project: die Zeile resources: ["interactions/*"] stehen, sonst kopiert Quarto die Dateien beim Render nicht in den docs/-Ordner mit.

Drei Wege, ein Spiel einzubauen

Variante Aufwand Vorteil Nachteil
A — Statisches HTML (eigenes Widget, wie oben) hoch zur Erstellung, niedrig danach volle Kontrolle, kein Konto, läuft offline erfordert HTML/JS-Kenntnisse oder KI-Hilfe
B — H5P (h5p.org / Moodle / WordPress) mittel viele fertige Vorlagen (Quiz, Drag-Drop, Memory) erfordert Konto, externe Abhängigkeit
C — externer Editor (Genially, Kahoot, Quizizz) niedrig schnell, hübsch, kollaborativ Werbung, externe Datenhaltung, oft Login nötig
Mach mit (Variante A — eigenes Spiel)

Schauen Sie sich interactions/lernspiel-kapitel1.html in RStudio an (im Files-Tab anklicken → Open in editor). Es ist eine einzige HTML-Datei mit eingebetteten Styles und JavaScript — kein Backend, kein Build-Schritt. Diesen Bauplan können Sie selbst (oder Ihr KI-Tutor) für ein eigenes Spiel im gewählten Strang nutzen.

Strang-Beispiele:

  • Strang A — Drag-and-Drop: „Welche Aussage ist eine Hypothese, welche eine Behauptung?”
  • Strang B — Multiple Choice: „Welche der drei Quellen ist primärwissenschaftlich?”
  • Strang C — Memory: „Methodenbegriff zu transparenter Beschreibung”
Variieren Sie
  1. iFrame-Höhe ändern: Setzen Sie height von 900 auf 600 herab und beobachten Sie, was abgeschnitten wird. Wann ist 90 % Bildschirmhöhe (style="height:90vh") die bessere Wahl?
  2. Eigenes Mini-Spiel: Lassen Sie den KI-Tutor ein einfaches Drag-and-Drop-Quiz zu Ihrem Strang als HTML-Datei generieren (Prompt aus dem Anhang nutzen). Speichern Sie es in interactions/ und betten Sie es ein.
  3. Eigener System-Prompt: Schreiben Sie einen System-Prompt für einen GPT, der ein bestimmtes Konzept aus Ihrem Strang erklärt. Drei Sätze: Rolle, Verhalten, Grenzen.
  4. iFrame versus Link: Bauen Sie eine Seite ein, die X-Frame-Options: DENY setzt (z. B. https://google.com). Was sehen Sie? Wann ist ein einfacher Link die bessere Lösung als ein iFrame?

H5P-Embed (Variante B, optional)

H5P liefert fertige interaktive Vorlagen, die Sie ohne Programmierung im Browser konfigurieren. Workflow:

  1. Konto auf https://h5p.org oder https://h5p.com anlegen.
  2. „New Content” → Drag the Words (oder eine andere Vorlage) wählen.
  3. Mini-Quiz zu drei Begriffen aus Ihrem Strang bauen.
  4. Auf Embed klicken, den iFrame-Code kopieren.
  5. In Ihr .qmd-Kapitel einfügen — fertig.

Ein H5P-Embed sieht typischerweise so aus:

<iframe src="https://h5p.org/h5p/embed/617"
        width="100%" height="500" frameborder="0"
        allowfullscreen="allowfullscreen">
</iframe>

Vorteil: H5P kümmert sich um die Speicherung der Antworten, falls Sie LRS-Anbindung haben. Nachteil: externe Abhängigkeit, Konto nötig, Werbung in der Free-Variante.

KI-Tutor als Link

Der Tutor läuft nicht im iFrame, sondern in einem zweiten Tab. Sie legen einen GPT (oder Custom GPT) mit System-Prompt an, holen sich den öffentlichen Link, bauen ihn als Markdown-Link an passende Stellen im Skript ein. Drei Hosting-Varianten zur Auswahl:

Option 1 — Custom GPT auf ChatGPT (heute genutzt)

Einfachste Variante, weltweit verfügbar, kein Hochschul-Login nötig. Im Skript einbauen:

[Fragen Sie den Skript-Tutor →](https://chatgpt.com/g/g-69fec93348108191af1ffb8ce0cf6712-tutor-interaktive-skripte){target="_blank"}

Der vorbereitete Tutor für diesen Workshop liegt unter dem oben verlinkten URL — der System-Prompt steckt dort schon drin. Die Lernenden klicken, ChatGPT-Login öffnet sich (kostenlos), der Tutor antwortet sofort im Workshop-Kontext.

Datenschutz: ChatGPT speichert Konversationen auf US-Servern. Für offene Lehre ohne personenbezogene Daten unkritisch. Wenn Sie europäische Datenhaltung brauchen, nehmen Sie Option 2.

Option 2 — Custom GPT in der AcademicCloud

Variante mit europäischer Datenhaltung und Hochschul-Login. Sie legen einen GPT mit dem vorbereiteten System-Prompt an (siehe Anhang Tutor-Prompt) und verlinken ihn:

[Fragen Sie den Skript-Tutor →](https://chat.academiccloud.de/chat/IHR-LINK){target="_blank"}

Lernende loggen sich mit Hochschul-Credentials ein.

Option 3 — eigenes statisches Widget oder voll gehosteter Tutor

Für Fortgeschrittene: ein HTML-Widget, das eine OpenAI-kompatible API direkt anspricht (Vorbild: _relevance-widget.qmd in https://github.com/TH-Koln-Bartnik/genai4teaching), oder eine eigene Web-App auf AWS Amplify mit RAG über Skriptinhalte (Vorbild: BWL1 Tutor-Chatbot). Mollick und Mollick (2023) geben einen Überblick über die didaktischen Designentscheidungen.

Mach mit

Wir nutzen heute den vorbereiteten Workshop-Tutor auf ChatGPT — Sie müssen keinen eigenen GPT bauen, sondern nur den Link einbauen:

[Fragen Sie den Skript-Tutor →](https://chatgpt.com/g/g-69fec93348108191af1ffb8ce0cf6712-tutor-interaktive-skripte){target="_blank"}

Setzen Sie diesen Link an passende Stelle in Ihr Skript (z. B. in einen Callout-Tip am Kapitelende). Rendern Sie und prüfen Sie den Klick.

Wer einen eigenen Tutor mit eigenem System-Prompt bauen will, springt zum Tutor-Prompt-Anhang und legt sich Option 1 oder Option 2 selbst an.

Wann kein KI-Tutor?

Drei Situationen, in denen Sie es lassen sollten:

  • Klar definierte Faktenfragen — wenn die Antwort eindeutig ist und in einem Lehrbuch steht, ist ein Quiz besser.
  • Sensible persönliche Themen — der Tutor soll Lernen begleiten, keine Beratung ersetzen.
  • Zeitkritisches Pflichtwissen — wenn die Lernenden in zehn Minuten etwas können müssen, ist Frontalanleitung effizienter als ein Gespräch (Kasneci et al., 2023).
Was, wenn …
  • iFrame zeigt nichts an? → Pfad prüfen: aus parts/05-iframes-tutor.qmd zeigt ../interactions/datei.html ins richtige Verzeichnis. Externe URLs immer mit https:// beginnen. Manche Seiten erlauben kein Embedding (HTTP-Header X-Frame-Options) — dann nur als Link verwenden.
  • iFrame leer im gerenderten docs/-Ordner? → In _quarto.yml muss resources: ["interactions/*"] unter project: stehen, damit Quarto die HTML-Dateien beim Render mitkopiert.
  • iFrame zu kurz, Inhalt abgeschnitten? → height höher setzen, oder style="height: 90vh" für 90 % der Bildschirmhöhe.
  • AcademicCloud-Link funktioniert für andere nicht? → Sharing aktivieren in den GPT-Einstellungen, „öffentlich” oder „mit Link”.
  • Tutor antwortet auf alles, auch außerhalb des Themas? → System-Prompt nachschärfen, explizit Grenzen setzen („Antworten Sie nur zu Quarto-Workflow-Fragen”).
