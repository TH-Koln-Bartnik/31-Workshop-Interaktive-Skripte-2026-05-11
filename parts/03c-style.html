5 — TH-Köln-Stil einbauen

Kür-Kapitel der Phase 3

Dieses Kapitel ist Auftakt der Kür. Wer eher zur Veröffentlichung will, springt direkt zu Kapitel 8 (GitHub Pages); Stil ist auch im Nachgang einbaubar, sobald Ihr Skript einmal live ist.

Was Sie nach diesem Kapitel können

  • die Hochschulfarben über _brand.yml setzen,
  • per CSS scharfkantige Callouts, Tabellen und Code-Blöcke gestalten,
  • das TH-Köln-Logo einbinden und prüfen, dass es überall korrekt erscheint.

Worum es im Kern geht

Gestaltung ist nicht Dekoration, sondern Lese-Hilfe. Ein konsistentes Layout reduziert kognitive Last (Mayer, 2021) — die Lernenden müssen nicht für jedes neue Element wieder verstehen, was wo wichtig ist. Tufte (2001) formuliert dasselbe Prinzip für Datenvisualisierung: maximaler Informationsgehalt bei minimalem Tinte-zu-Daten-Verhältnis. Beides spricht für Zurückhaltung im Stil — wenige Farben, klare Hierarchien, kein Schmuck.

Wie Quarto Aussehen handhabt

Drei Stellschrauben, in dieser Reihenfolge:

  1. _brand.yml — die Hochschulfarben und -schriften an einem Ort.
  2. styles.scss — alles, was _brand.yml nicht abdeckt (Layout, Akzente, eigene Boxen).
  3. format.html.theme in _quarto.yml — wählt das Bootstrap-Basistheme.

Die TH-Köln-Palette in Stichworten

Drei Farben, sonst nur Grautöne. Alles andere ist Verstoß gegen das Corporate Design.

Rolle Hex Wo?
TH Red #c81e0f Links, Akzente, wichtige Hervorhebungen
TH Magenta #b43092 Strukturlinien, Callout-Ränder
TH Orange #ea5a00 Sehr sparsam — Warnungen, Verlauf
Ink #111111 Fließtext (weicheres Schwarz)
Paper #ffffff Hintergründe, immer

Mach mit — _brand.yml kopieren

Im Repository des Workshops liegt eine fertige _brand.yml mit der TH-Köln-Palette. Kopieren Sie sie in Ihr Projektverzeichnis. Beim nächsten quarto render sind Links rot, Überschriften schwarz, Akzente magenta.

color:
  palette:
    th-red: "#c81e0f"
    th-magenta: "#b43092"
    th-orange: "#ea5a00"
  primary: th-red
  secondary: th-magenta

SCSS-Ergänzungen

Drei Dinge, die _brand.yml nicht regelt: scharfe Ecken („kantig”), die magenta H2-Unterstreichung und die TH-Farbleiste am Seitenanfang. Quarto verarbeitet eigene Stile als SCSS (Sass) — eine Erweiterung von CSS, die mit Bootstrap zusammenarbeitet. Sie brauchen dafür kein Sass-Wissen: jede gültige CSS-Regel ist auch gültiges SCSS.

Die Datei muss styles.scss heißen (nicht .css) und am Anfang einen Layer-Marker enthalten — sonst bricht quarto render mit einer Fehlermeldung „doesn’t contain at least one layer boundary” ab.

Legen Sie styles.scss an und binden Sie sie in _quarto.yml ein:

format:
  html:
    theme:
      - cosmo
      - styles.scss

In styles.scss (Auszug — Layer-Marker am Anfang ist Pflicht):

/*-- scss:rules --*/

:root {
  --th-red: #c81e0f;
  --th-magenta: #b43092;
}

h1, h2, h3 { border-radius: 0 !important; font-weight: 600; }
h2 { border-bottom: 2px solid rgba(180, 48, 146, 0.35); padding-bottom: 0.3rem; }
.callout { border-radius: 0 !important; }
pre, code { border-radius: 0 !important; }
Mach mit — Vorher/Nachher

Rendern Sie Ihr Skript einmal vor und einmal nach dem Einbau von _brand.yml. Vergleichen Sie beide Versionen nebeneinander. Was ändert sich? Was nicht? Schreiben Sie sich drei Beobachtungen auf — wir nutzen sie im Showcase am Ende.

Variieren Sie
  1. Andere Hochschulfarbe: Tauschen Sie TH Red (#c81e0f) gegen die Hauptfarbe Ihrer eigenen Fakultät oder Hochschule. Render — wie wirkt der Wechsel?
  2. Eigene CSS-Regel: Ergänzen Sie in styles.scss eine Regel, die <strong>-Text in TH Magenta hervorhebt (strong { color: var(--th-magenta); }). Render und prüfen.
  3. Anderes Theme: Tauschen Sie unter format.html.theme cosmo gegen ein anderes Bootstrap-Theme (z. B. flatly, journal, cerulean, simplex) und vergleichen Sie das Gesamtbild.
  4. Schriftgröße erhöhen: Setzen Sie in styles.scss body { font-size: 1.1rem; }. Render — wie verändert sich Lesbarkeit und Folgewirkung auf andere Elemente?

Logo einbinden

  1. Laden Sie das TH-Köln-Logo (PNG, transparent) in images/th-koeln-logo.png.
  2. In _brand.yml:
logo:
  small: images/th-koeln-logo.png
  medium: images/th-koeln-logo.png
  1. Rendern und prüfen.
Was, wenn …
  • Farben tauchen nicht auf? → Cache leeren (quarto render --cache-refresh) oder den Ordner _freeze/ löschen.
  • CSS wird nicht geladen? → Pfad in _quarto.yml prüfen, relative Pfade ohne führenden Slash.
  • Logo wird nicht angezeigt? → PNG-Datei muss tatsächlich im images/-Ordner liegen, nicht nur in _brand.yml referenziert sein.
  • Farben sehen ausgewaschen aus? → Eventuell ein Bootstrap-Theme aktiv, das die Variablen überschreibt. theme: [cosmo, styles.scss] in dieser Reihenfolge halten — die SCSS-Datei muss zuletzt geladen werden.
  • „doesn’t contain at least one layer boundary” beim Render? → Am Anfang von styles.scss muss /*-- scss:rules --*/ stehen.

Reaktivierung: Aktiv auslaufen lassen

Schieber bewegen, Stil-Aspekte einzeln an- und abschalten. Links der TH-Köln-Stil, rechts der Quarto-Default. Welche der drei Stellschrauben (Farbe, Schrift, Kanten) verändert das Bild am stärksten?
