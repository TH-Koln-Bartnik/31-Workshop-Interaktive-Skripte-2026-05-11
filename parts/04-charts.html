6 — Interaktive Diagramme

Was Sie nach diesem Kapitel können

  • ein interaktives Diagramm mit echten Daten in Ihr Skript einbauen,
  • Daten direkt aus dem Netz laden (z. B. Our World in Data),
  • erklären, wann sich Interaktivität didaktisch lohnt und wann nicht.

Worum es im Kern geht

Diagramme tun in einem Skript zwei Dinge auf einmal — sie zeigen Daten und sie laden zur Interpretation ein. Tufte (2001) formuliert die Maxime: maximale Information bei minimalem Schmuck. Healy (2018) ergänzt für die digitale Lehre: Interaktivität rechtfertigt sich nur, wenn die Lernenden tatsächlich etwas mit ihr anfangen — verschieben, ein- und ausblenden, vergleichen. Andernfalls reicht ein PNG.

Im Skript unten sehen Sie zwei Familien interaktiver Diagramme nebeneinander: Plotly.js-Widgets (oben, mit Hover, Zoom, Reihen ein- und ausblenden) und Observable JS (weiter unten, mit eigenen reaktiven Steuerelementen wie Slider und Dropdown). Beide rendern ohne lokale R-Installation. Wer R lokal hat, schreibt die Plotly-Charts auch in R; wer nicht, kopiert die HTML-Variante eins zu eins aus dem Quelltext.

Vier Wege, ein Diagramm einzubauen

Weg Werkzeug Wann passend?
R + plotly R-Code-Chunk in Quarto Sie kennen R oder wollen es lernen. Volle Kontrolle, schöne Grafiken (Wickham et al., 2019).
Plotly.js direkt HTML + JavaScript Sie haben kein R, aber wollen Plotly-Charts. Funktioniert in jedem Browser.
Observable JS + Plot {ojs}-Chunk in Quarto Sie wollen reaktive Steuerelemente (Slider, Dropdown), die das Diagramm live verändern — ohne JavaScript-Schreiben.
iFrame mit Datawrapper / Flourish externer Editor, Embed-Code Sie wollen gar nicht programmieren.

Observable JS — Reaktivität ohne JavaScript-Schreiben

Plotly liefert Interaktivität, die die Bibliothek vorgibt: Hover, Zoom, Reihen ein- und ausblenden. Was Plotly nicht liefert, sind eigene Steuerelemente — ein Slider, der den Datenausschnitt verändert, ein Dropdown, das die Farbgruppe wählt, eine Checkbox-Liste, die Reihen filtert. Genau dafür gibt es Observable JS (kurz OJS), eine Programmierumgebung im Browser, die Quarto direkt eingebaut hat. In einfachen Worten: Sie schreiben in einen {ojs}-Block ein paar Zeilen, die Quarto in ein klickbares Bedienfeld plus reagierendes Diagramm verwandelt — ohne dass Sie JavaScript lernen müssen. Die Bibliothek Observable Plot ist mit dabei und macht Diagramme mit ähnlich wenig Code wie ggplot2.

Mehr zur Idee von Observable (Observable, Inc., 2024); dieselben Reaktivitäts-Prinzipien stecken hinter Shiny in R (Chang et al., 2023), nur ohne den R-Server.

Ein-Slider-Demo — die kleinste reaktive Geschichte

Ein Schieberegler oben, ein Diagramm unten. Schieben Sie den Regler — die Balken passen sich live an.

viewof bis_jahr = Inputs.range([1990, 2023], {label: "Daten bis Jahr", step: 1, value: 2023})

publikationen = [
  {jahr: 1990, anzahl: 0.8},
  {jahr: 1995, anzahl: 1.0},
  {jahr: 2000, anzahl: 1.2},
  {jahr: 2005, anzahl: 1.6},
  {jahr: 2010, anzahl: 2.1},
  {jahr: 2015, anzahl: 2.5},
  {jahr: 2020, anzahl: 2.9},
  {jahr: 2023, anzahl: 3.3}
]

gefiltert = publikationen.filter(d => d.jahr <= bis_jahr)

Plot.plot({
  marks: [
    Plot.barY(gefiltert, {x: "jahr", y: "anzahl", fill: "#c81e0f"}),
    Plot.ruleY([0])
  ],
  y: {label: "Publikationen (Mio. pro Jahr)"},
  x: {label: "Jahr"},
  marginLeft: 60,
  height: 360
})

Was passiert hier? Drei Bausteine: Inputs.range(...) baut den Schieberegler; viewof bis_jahr = ... macht aus dem Reglerwert eine Variable, auf die andere Blöcke reagieren; das Plot.plot(...)-Stück zeichnet das Diagramm immer neu, wenn bis_jahr sich ändert. Wenn Sie das verstanden haben, können Sie dieses Muster auf jeden Datensatz anwenden.

Mach mit — eigenes Observable-Diagramm

Übernehmen Sie den Code des Strangs Ihrer Wahl in Ihr eigenes Kapitel — kopieren Sie den ganzen {ojs}-Block samt umgebendem content-visible-Wrapper. Tauschen Sie die Daten in le_daten/pub_daten/prereg_daten gegen drei bis fünf Werte aus Ihrem Lehrgebiet. Render mit Strg+Shift+B und probieren Sie das Steuerelement im Browser.

Variieren Sie
  1. Steuerelement tauschen: Ersetzen Sie Inputs.range(...) durch Inputs.select([...], {label: "Wahl"}) (Dropdown) oder Inputs.radio([...], {label: "Wahl"}) (Radio-Buttons). Welches Bedienelement passt zu welcher Wahl?
  2. Zweites Steuerelement koppeln: Bauen Sie einen zweiten viewof-Block mit einer zweiten Variable und kombinieren Sie beide im filter-Schritt — z. B. „Land” und „Jahr ab” gleichzeitig.
  3. Plot-Typ wechseln: Tauschen Sie Plot.line(...) gegen Plot.barY(...) oder umgekehrt. Wann wird die Botschaft klarer?
  4. Aus CSV laden: Statt die Daten im Code anzulegen, lesen Sie eine CSV ein — daten = FileAttachment("meine-daten.csv").csv({typed: true}). Datei in data/ ablegen und im Quarto-YAML als Resource registrieren.

Plot-Bibliothek-Spickzettel. Plot.line(...), Plot.barY(...), Plot.dot(...), Plot.area(...), Plot.text(...), Plot.ruleY([...]) (horizontale Linie), Plot.ruleX([...]) (vertikale Linie), Plot.cell(...) (Heatmap-Kachel). Vollständig: https://observablehq.com/plot/marks/.

Wann kein interaktives Diagramm?

  • Reine Illustration — wenn die Lernenden nichts ändern sollen, sondern nur etwas „sehen”, reicht ein Bild (Tufte, 2001).
  • Print-Version wichtig — interaktive Plots werden im PDF zu statischen Screenshots.
  • Performance — sehr große Datensätze (>100.000 Punkte) machen plotly träge.
Mach mit

Tauschen Sie im Code zwei Datenpunkte gegen welche aus, die für Ihren Strang interessant sind. Lokal rendern und im Browser ansehen. (Pushen folgt am Tagesende in Kap. 8.)

Variieren Sie
  1. Datenpunkt austauschen: Ändern Sie im <script>-Block einen Y-Wert deutlich (z. B. setzen Sie das Jahr 2022 auf 90 statt 80). Rendern Sie und beobachten Sie, wie Plotly die Y-Achse automatisch skaliert.
  2. Diagrammtyp wechseln: Tauschen Sie type: 'bar' gegen type: 'scatter' mit mode: 'lines+markers'. Render und entscheiden Sie: Welche Daten passen besser zu welchem Typ?
  3. Vierte Reihe: Fügen Sie in Strang A eine vierte Datenreihe hinzu (z. B. ein weiteres Land, ein weiteres Bundesland) — vergeben Sie eine vierte Farbe. Render.
  4. Eigene Daten: Ersetzen Sie den Beispieldatensatz vollständig durch echte Daten aus Ihrem Lehrgebiet. Render und prüfen Sie, ob die Achsenbeschriftungen noch passen.
Was, wenn …
  • R-Render bricht ab mit there is no package called 'plotly' → entweder install.packages(c("plotly", "tibble")) in der R-Konsole, oder die HTML-Variante aus diesem Kapitel kopieren — kein R nötig.
  • Plotly-Chart erscheint nicht im Browser? → Prüfen, ob der CDN-Loader (<script src="https://cdn.plot.ly/...">) am Anfang des Kapitels eingebunden ist.
  • CSV lässt sich nicht laden → lokal in data/ speichern und von dort lesen (CORS).
  • Diagramm zu klein → height im <div style="..."> höher setzen.
